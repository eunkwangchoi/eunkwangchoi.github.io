스크리브너 윈도우 버전이 2025년 9월 3일 기준 v3.1.5에서 v3.1.6으로 개선되었습니다.

## 1. 주요 개선

#

### (1) 컴파일 

- 컴파일러가 저장 위치를 선택할 때 파일 이름을 제안하고, 프로젝트별로 사용한 마지막 파일 이름을 계속 기억하도록 수정

### (2) 아웃라이너

- 바인더, 아웃라이너 또는 코르크보드에서 항목을 마우스 오른쪽 버튼으로 클릭하면 단축 메뉴가 열리되, 연결된 편집기에 해당 항목이 로드되지 않도록 아웃라인 선택 동작을 개선

### (3) 편집기

- 스크리브닝에서 문서 포커스를 전환할 때 이전 선택 영역이 비활성화되도록 텍스트 선택 동작 수정 (이전에는 선택 영역이 활성화된 것처럼 보였지만 반응하지 않은 채 남아있었음)
- 인라인 주석 표시 방법 개선 ⇒ 특히 인라인 주석이 줄을 가로지르는 경우, 전체 선택 없이도 커서를 주석 내 어디에나 위치시킨 상태에서 텍스트 색상 도구를 사용하여 색상을 변경할 수 있도록 수정
- **스타일 표시/숨김** 기능을 대부분의 워드 프로세서에서 표준으로 채택된 동작에 맞춰 토글할 수 있도록 수정

### (4) 기타

- RTF 형식 **스크래치패드 노트**의 <mark>[[문서 링크]]</mark> 자동 감지 기능이 활성화된 상태에서 위키 스타일 링크를 입력할 때 발생하던 충돌 현상 수정
- 공백 문자만으로 구성된 사용자 정의 텍스트 기반 아이콘 사용 시 발생하는 충돌 현상 수정
- 폴더 또는 파일 그룹 이름 끝에 붙은 불필요한 공백이 내보내기 시 제거되지 않아 깨진 디렉터리(파일 탐색기에서 삭제나 이름 변경을 할 수 없는)가 생성되던 버그 수정

## 2. 세부 개선

#

### (1) 컴파일

- 멀티마크다운을 이용해 ODT 및 FODT 파일을 컴파일하는 **MMD OpenOffice 문서** 컴파일 형식에서 색인 용어를 그룹화되거나 계층화 할 수 있도록 수정 (사용자 매뉴얼 부록 D5.6 참조)
- 마크다운이나 일반 텍스트로 컴파일 시 중복 이미지가 여러 번 내보내지지 않도록 수정

### (2) 아웃라이너

- 프로젝트 검색 필드에 포커스가 있을 때 위쪽 화살표 및 아래쪽 화살표 키로 결과 목록을 탐색할 수 있으며, 탭 키로 포커스를 결과로 이동할 수 있도록 수정
- 사용자 지정 아이콘을 수정하거나 폴더와 파일 간에 유형을 바꿀 경우 항목의 수정 날짜가 업데이트되지 않도록 수정
- **문서 ▸ 아이콘 변경 ▸ 텍스트로 아이콘 만들기**<sup>Documents ▸ Change Icon ▸ Icon from Text…</sup> 를 사용하여 빈 아이콘(공백 입력)을 생성할 수 있도록 수정

### (3) 편집기

- 텍스트 편집기에서 새 문서를 생성할 때 편집기의 포커스 위치가 아닌 바인더 선택 항목을 기준으로 새 항목이 배치되던 버그 수정
- 스크리브닝 모드에서 **선택 위치로 이동**<sup>Jump to Selection</sup> 명령이 작동하지 않던 문제 수정
- 스크리브닝 세션에서 **보이지 않는 문자 표시**<sup>Show Invisibles</sup> 토글 시 텍스트가 수직 중앙 또는 가장 아래 스크롤 위치로 이동하는 버그 수정
- 스크리브닝 모드에서 사용할 수 없는 **페이지 보기**<sup>Page View</sup> 키보드 단축키가 활성화되어 있는 버그 수정
- 6.00인치 오른쪽 들여쓰기가 적용된 텍스트를 복사하여 붙여넣을 때 들여쓰기가 0으로 변경되던 버그 수정
- 텍스트를 오른쪽에서 왼쪽으로 선택할 때 선택 영역 바로 왼쪽 텍스트의 서식이 표시되던 텍스트 속성 표시 문제 수정
- **다음 찾기** 및 **이전 찾기** 사용 시 줄바꿈이 안정적으로 탐지되지 않던 버그 수정
- **서식으로 찾기**<sup>Find by Formatting</sup> 의 하이라이트 검색 기능을 수정하여, 특정 색상을 검색할 때 검색 항목을 건너뛰거나 실패할 수 있던 문제 해결
- 맞춤법 제안에서 사전 인코딩을 지원하지 않는 문자가 포함된 단어가 제외되도록 수정 (선택 시 잘못된 대체 단어가 사용될 수 있었음)
- 소문자로 학습된 단어도 대문자로 작성 시 인식되도록 수정
- 스냅샷 롤백 후 편집기 여백이 일시적으로 제거되어 텍스트가 편집기 가장자리까지 밀려나는 버그 수정

### (4) 기타

- 자동 종료 설정을 **옵션 - 일반**의 별도 탭으로 이동
- **단편 소설**<sup>Short Story</sup> 프로젝트 템플릿의 텍스트를 소폭 수정
- **편집 ▸ 글쓰기 도구 ▸ 사전에서 찾기**<sup>Edit ▸ Writing Tools ▸ Look Up in Dictionary.com</sup> 기능의 사전 링크 업데이트
